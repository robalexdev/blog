What is this?

Web servers provide a timestamp in the HTTP Date header field or in custom headers. This web page generates a clock from those values, providing an accurate time even when your system clock is set to an incorrect time.

UTC and UTC(NIST) are well-trusted time-references which provide extremely accurate timekeeping. You'd typically synchronize with these references using protocols like NTP.

This webpage shows an alternate method which synchronizes over HTTP. This is similar to time.gov/, although a different approach is used to measure HTTP response timing. While this alternate approach should provide better synchronization, the NIST reference time is more accurate.

UTC(GitHub Pages) takes the HTTP Date response header from a request to the GitHub Pages static web hosting service. The request targets a unique non-existent file, which should bypass caches and generate a fresh 404 Not Found response. Since the Date header is rounded down to a whole number of seconds, the accuracy is at most 500ms.

Why are the times different?

Unfortunately, the two time sources are not always synchronized with eachother. In testing, I've seen offsets by a full one second. Larger offsets seem possible.
